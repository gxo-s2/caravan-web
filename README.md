# 🚙 CaravanShare: 카라반 공유 플랫폼 MVP

CaravanShare는 카라반(캠핑카) 소유자(호스트)와 이용을 원하는 사용자(게스트)를 연결하는 공유 플랫폼 서비스의 최소 기능 제품(MVP) 구현 프로젝트입니다. **바이브코딩 실전 문제** 시나리오를 바탕으로 개발되었습니다.

---

## 1️⃣ 기능 설명 및 기술 스택 (Features & Tech Stack)

### 🚀 프로젝트 목표
* **CaravanShare** MVP의 핵심 비즈니스 로직(예약, 결제 등)을 **SOLID 원칙** 기반의 확장 가능한 아키텍처로 구현하고, 현대적인 사용자 인터페이스를 구축하는 것을 목표로 합니다.

### 🌐 기술 스택 요약

| 구분 | 주요 기술 | 역할 |
| :--- | :--- | :--- |
| **프론트엔드** | **Next.js** (App Router), **TypeScript**, **Tailwind CSS** | 반응형 웹 인터페이스, 서버와의 데이터 통신 |
| **백엔드** | **Node.js/Express**, **TypeScript**, **Prisma ORM** | RESTful API 설계, 비즈니스 로직 처리, 데이터베이스 연동 |
| **데이터베이스** | **PostgreSQL** | 영구 데이터 저장 및 관리 |

### 🔑 구현된 핵심 기능 (Phase 1: MVP)

| 핵심 기능 영역 | 상세 구현 내용 |
| :--- | :--- |
| **1. 사용자 관리** | • **회원가입/로그인:** 호스트(HOST)/게스트(GUEST) 역할 구분 및 인증/인가 (로컬 저장소 기반 임시 인증).<br>• **프로필 관리:** 이름, 연락처, 평가, 신원 확인 필드 구현.<br>• **사용자 신뢰도 시스템:** 평가(평점)를 통한 신뢰도 반영 시뮬레이션. |
| **2. 카라반 정보 관리** | • **카라반 등록/삭제:** 호스트가 카라반을 등록하고, 등록한 카라반을 삭제할 수 있는 기능.<br>• **정보 제공:** 수용 인원, 편의시설, 사진, 위치 등 상세 정보 포함.<br>• **검색/조회:** 게스트가 카라반을 검색/조회하고 상태(사용가능/예약됨/정비중)를 확인할 수 있음. |
| **3. 예약 시스템** | • **예약 신청:** 게스트의 예약 신청 및 예약 번호 조회 기능.<br>• **호스트 관리:** 호스트는 **예약 목록**을 확인하고, 신청된 예약을 **승인/거절** 처리.<br>• **날짜 관리:** 예약 날짜 관리 (캘린더 연동 시뮬레이션) 및 **중복 예약 방지 로직** 구현. |
| **4. 결제 및 가격** | • **가격 설정:** 호스트가 일일 요금을 설정.<br>• **가격 계산:** 렌탈 기간 기반으로 총 가격을 계산하는 로직.<br>• **결제 시스템:** 선결제 시스템 시뮬레이션 및 결제 이력 조회 기능. |
| **5. 리뷰/평가** | • **리뷰 작성:** 거래 완료 후 게스트가 리뷰 작성 (평점 시스템 1-5점).<br>• **후기 표시:** 카라반 상세 페이지에 해당 후기 목록 표시. |

## 2️⃣ 프로젝트 실행 방법 (실습 환경 가이드)

프로젝트의 모든 기능을 확인하기 위해서는 **백엔드 서버(API)**와 **프론트엔드 서버(웹사이트)** 두 개를 동시에 실행해야 합니다.

### 2.1. 개발 환경 준비

1.  **Repository 클론:** 이 프로젝트를 로컬 환경(VS Code 등)으로 복사합니다.
    ```bash
    git clone [깃허브 저장소 주소]
    cd caravan-app
    ```
2.  **의존성 설치:** `backend`와 `frontend` 폴더 각각에서 의존성을 설치합니다.
    ```bash
    # 백엔드 폴더에서
    cd backend
    npm install
    
    # 프론트엔드 폴더로 이동
    cd ../frontend
    npm install
    ```

### 2.2. 백엔드 서버 실행 (API 서버)

백엔드 서버를 먼저 실행하여 프론트엔드에서 데이터를 요청할 수 있도록 준비합니다.

1.  **데이터베이스 마이그레이션:** 데이터베이스(`PostgreSQL`)가 준비되었다고 가정하고 스키마를 적용합니다.
    ```bash
    cd ../backend
    npx prisma migrate dev --name init_project
    ```
2.  **서버 실행:**
    ```bash
    npm run dev
    # 서버는 http://localhost:3001 에서 실행됩니다.
    ```

### 2.3. 프론트엔드 서버 실행 (웹사이트)

새로운 터미널을 열어 프론트엔드 서버를 실행합니다.

1.  **프론트엔드 폴더로 이동:**
    ```bash
    cd frontend
    ```
2.  **서버 실행:**
    ```bash
    npm run dev
    # 웹사이트는 http://localhost:3000 에서 실행됩니다.
    ```

### 2.4. 웹사이트 접속 및 테스트

두 서버 (**`http://localhost:3001`** 및 **`http://localhost:3000`**)가 모두 실행된 후, 웹 브라우저에서 아래 주소로 접속하여 모든 기능을 테스트할 수 있습니다.

* **개발 환경 접속 주소:** 👉 **`http://localhost:3000`**
* **배포된 웹사이트 접속 주소:** (선택 사항) 👉 **`https://caravan-app-chi.vercel.app`**
